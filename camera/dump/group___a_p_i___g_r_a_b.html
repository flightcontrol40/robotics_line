◆ CameraGetImageBuffer()

MVSDK_API CameraSdkStatus __stdcall CameraGetImageBuffer ( CameraHandle  hCamera,
tSdkFrameHead *  pFrameInfo,
BYTE **  pbyBuffer,
UINT  wTimes 
)

Get a frame of image data. To improve efficiency, the SDK uses a zero-copy mechanism for image capture. This function actually obtains a buffer address in the kernel.

Parameters
[in]hCameraCamera handle.
[out]pFrameInfoThe header information pointer of the image.
[out]pbyBufferReturns the buffer pointer of the image data.
[in]wTimesTimeout for grabbing an image in milliseconds. The function returns a timeout error if no image has been obtained within wTimes.
Returns
Returns CAMERA_STATUS_SUCCESS(0) successfully. Otherwise, it returns a non-zero error code. Please refer to the definition of the error code in CameraStatus.h.
Note
After the function is successfully called, CameraReleaseImageBuffer must be called to release the buffer so that the kernel can continue to use the buffer.
Examples:
grab.cpp, roi.cpp, and soft_trigger.cpp.

◆ CameraSnapToBuffer()

MVSDK_API CameraSdkStatus __stdcall CameraSnapToBuffer ( CameraHandle  hCamera,
tSdkFrameHead *  pFrameInfo,
BYTE **  pbyBuffer,
UINT  wTimes 
)

Take an image into the buffer. The camera will enter snap mode and automatically switch to snap mode resolution for image capture.

Parameters
[in]hCameraCamera handle.
[out]pFrameInfoThe header information pointer of the image.
[out]pbyBufferReturns the buffer pointer of the image data.
[in]wTimesTimeout for grabbing an image in milliseconds. The function returns a timeout error if no image has been obtained within wTimes.
Returns
Returns CAMERA_STATUS_SUCCESS(0) successfully. Otherwise, it returns a non-zero error code. Please refer to the definition of the error code in CameraStatus.h.
Note
After the function is successfully called, CameraReleaseImageBuffer must be called to release the buffer so that the kernel can continue to use the buffer.
Warning
This function may switch the resolution, so the efficiency will be lower than CameraGetImageBuffer. If you do not need to switch resolution capture, use CameraGetImageBuffer.
