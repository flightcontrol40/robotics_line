Save the image buffer data as a picture file.

Parameters
[in]hCameraCamera handle.
[in]lpszFileNameThe picture saves the full path to the file.
[in]pbyImageBufferImage data buffer.
[in]pFrInfoThe frame header information of the image.
[in]byFileTypeImage save format. See the definition of emSdkFileType for the range of values.
[in]byQualityThe quality factor of the saved image. This parameter is valid only when saving in JPG format. The range is from 1 to 100. The rest of the format can be written as 0.
Returns
Returns CAMERA_STATUS_SUCCESS(0) successfully. Otherwise, it returns a non-zero error code. Please refer to the definition of the error code in CameraStatus.h.
Note
Currently supports BMP, JPG, PNG, RAW four formats. Among them, RAW represents the raw data output by the camera. Saving RAW format files requires pbyImageBuffer and pFrInfo to be obtained by CameraGetImageBuffer, and without CameraImageProcess converting to BMP format; otherwise, if you want to save to BMP JPG or PNG format, pbyImageBuffer and pFrInfo are RGB format data processed by CameraImageProcess. Specific usage can refer to Advanced's routines.
Examples:
grab.cpp, grab_ex.cpp, roi.cpp, and soft_trigger.cpp.
